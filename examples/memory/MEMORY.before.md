- workflow step 1: run a command and check result.
- workflow step 2: run another command and compare output.
- workflow step 3: write a report to a local path.
- workflow step 4: copy exact IDs into a long-term note.
- workflow step 5: repeat the command every week.
- workflow step 6: keep raw operational details forever.
- workflow step 7: include commit SHA, releaseId, artifact hash, cron job ID.
- workflow step 8: store every procedure in MEMORY instead of a skill.
